1. GitHubのアカウント取得　https://github.com/
2. GitHubにてリポジトリを作成する(これが案件ごとのバージョン管理となるフォルダ)
3. SourceTreeのダウンロード


——ここからがローカルでのリポジトリ作業

1. リモートでGitHubアカウントを接続 
(ホスト：GitHub / ユーザ名：GitHubのユーザ名 / パスワード：GitHubのパスワード / プロトコル：HTTPS)
2. 2で作ったものが出てくるのでクローンをクリック（クローンは、ファイルを丸ごと複製し、ローカル環境へダウンロードすること）
3. ローカルにクローンフォルダをダウンロードできました。
4. 管理したいファイルを作成した6のフォルダに保存します。
5. SourceTreeを開くと保存したファイルが表示されているので、ファイルを選択し右クリックで「Indexに追加」を選択します。
6. ファイルを「Indexにステージしたファイル」の項目に移動しました。
7. 次に上にある「コミット」ボタンをクリックします。テキストエリアは任意で入力
8. コミットするとアップされたことになり、左上の履歴ボタンから確認ができます。


——ここからがリモートでのリポジトリ作業

1. SourceTreeの右上にある「プッシュ」ボタンをクリックするとプッシュに必要な情報の入力を求められます。
2. アップしたいファイルにチェックを入れ、OKボタンを押します。
3. プッシュが出来たかどうかGitHubで確認してみます。 https://github.com/tatsuyamizuguchi/ (マイページ)
4. ファイルがアップされていたら成功です。


——ファイル内容変更のテスト

1. アップしたファイルの内容をローカルフォルダで変更してみます。
2. SourceTreeを開くと変更したファイルが緑背景で変更された箇所が表示されます。
3. ファイルをコミットします。
4. プッシュでリモートにアップします。(上書き)


——ファイルを戻したい時のテスト

1. ファイル内容を変更します。
2. SourceTreeを開くと変更したファイルが緑背景で変更された箇所が表示されます。
3. リセットボタンを押すと変更履歴がでるので、戻りたいバージョンに戻します。



——GitHub上のファイルを最新版にしたい時のテスト

1. GitHub上で変更したファイルを作ってみます。
2. SourceTreeでプルというボタンがあるので、クリックするとローカルファイルと同期します。
